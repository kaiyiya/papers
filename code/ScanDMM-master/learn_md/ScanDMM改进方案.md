## 一、研究背景与问题分析

### 1.1 ScanDMM现状

#### 当前架构

```
┌─────────────────────────────────────────────────────────┐
│                    ScanDMM 当前架构                       │
└─────────────────────────────────────────────────────────┘

输入：360度图像 I (3, H, W)
  │
  ▼
┌─────────────────────┐
│   Sphere CNN        │  ← 一次性提取全局特征（100维）
│   (全图处理)        │     计算成本：100%
└─────────────────────┘
  │
  ▼
┌─────────────────────┐
│   状态初始化        │  ← z_0 = F(z_0, x_1)
│   z_0               │
└─────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────┐
│           固定长度序列生成 (T_max步)            │
│  ┌─────────────────────────────────────────┐  │
│  │ for t in range(1, T_max+1):            │  │
│  │   z_t = GatedTransition(z_{t-1}, s)   │  │ ← 所有t使用相同的s
│  │   x_t = Emitter(z_t)                  │  │
│  └─────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
  │
  ▼
输出：扫描路径 x_{1:T_max}
```

#### 核心问题

```
┌─────────────────────────────────────────────────────────┐
│                    核心问题分析                          │
└─────────────────────────────────────────────────────────┘

问题1：静态特征提取
  ❌ 所有时间步使用相同的全局特征
  ❌ 无法根据注视点动态调整关注区域
  ❌ 简单场景和复杂场景使用相同计算资源

问题2：固定计算成本
  ❌ 无论场景复杂度，都执行相同的CNN前向传播
  ❌ 计算成本 = 固定CNN成本 + 序列生成成本
  ❌ 无法自适应调整

问题3：固定长度序列
  ❌ 所有扫描路径都是固定长度T_max
  ❌ 简单场景浪费计算，复杂场景可能不够
  ❌ 缺乏主动决策机制

问题4：忽略360度图像特殊性
  ❌ 没有考虑等距圆柱投影的失真
  ❌ 没有考虑球面几何特性
  ❌ 边界处理可能出错
```

## 二、AdaptiveNN核心思想启发

### 2.1 AdaptiveNN的"从粗到细"机制

```
┌─────────────────────────────────────────────────────────┐
│              AdaptiveNN 核心机制示意                      │
└─────────────────────────────────────────────────────────┘

阶段1：Glance（快速扫描）
  ┌──────────────┐
  │ 降采样图像   │  ← 低分辨率，快速获取全局概览
  │ 粗粒度特征   │     计算成本：20-30%
  └──────────────┘
        │
        ▼
阶段2：Fixations（注视序列）
  ┌─────────────────────────────────────┐
  │  注视1: 选择感兴趣区域 → 精细观察  │
  │  注视2: 选择感兴趣区域 → 精细观察  │
  │  注视3: 选择感兴趣区域 → 精细观察  │
  │  ...                                │
  │  每次注视：局部高分辨率处理         │
  │  计算成本：每次5-10%                │
  └─────────────────────────────────────┘
        │
        ▼
阶段3：决策终止
  ┌──────────────┐
  │ 价值网络评估 │  ← 信息量足够？继续/停止
  │ 主动结束     │
  └──────────────┘

总计算成本 = 20-30% + N × 5-10%
  - 简单场景：N=2-3 → 总成本 30-50%
  - 复杂场景：N=8-10 → 总成本 60-80%
```

### 2.2 关键创新点

```
┌─────────────────────────────────────────────────────────┐
│                  AdaptiveNN 关键创新                      │
└─────────────────────────────────────────────────────────┘

1. 动态资源分配
   ✅ 初始阶段：低分辨率/粗粒度特征（快速）
   ✅ 后续阶段：高分辨率/细粒度特征（详细）
   ✅ 资源消耗取决于注视次数，而非场景复杂度

2. 顺序注视策略
   ✅ 从粗到细：先看全局，再看局部
   ✅ 自适应：根据任务难度动态调整注视次数
   ✅ 聚焦：只处理任务相关的区域

3. 自适应推断成本
   ✅ 可通过调节阈值在线调整平均计算成本
   ✅ 无需重新训练即可适配不同资源约束

4. 自奖励强化学习
   ✅ 将任务损失作为奖励信号
   ✅ 端到端训练，无需额外标注
```

## 附录：关键公式

### A.1 自适应融合公式

```
s_fused = α(t) · s_coarse + (1 - α(t)) · s_fine

其中：
  α(t) = {
    0.8,  if t < 0.3·T_max      (粗粒度阶段)
    f(t), if 0.3·T_max ≤ t < 0.7·T_max  (过渡阶段)
    0.2,  if t ≥ 0.7·T_max      (细粒度阶段)
  }
```

### A.2 球面距离公式

```
大圆距离（球面距离）：
  d = R · arccos(sin(φ₁)sin(φ₂) + cos(φ₁)cos(φ₂)cos(Δλ))

其中：
  R: 球面半径
  φ₁, φ₂: 两个点的纬度
  Δλ: 经度差
```

### A.3 返回抑制公式

```
抑制权重：
  w_inhibit(x, y) = exp(-d² / (2σ²))

其中：
  d: 到已访问注视点的距离
  σ: 抑制范围参数
```
