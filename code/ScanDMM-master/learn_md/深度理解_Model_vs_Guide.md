## 📖 第一步：用生活例子理解核心思想

### 场景：预测明天的天气

假设你要预测明天的天气，有两种方式：

---

### 方式1：Model（生成模型）- "普通人预测"

**特点**：只能看过去和现在

```
你现在知道：
- 今天天气：晴天
- 今天的温度：25度
- 你看到：天空有很多云（图像特征）

你预测明天：
- 根据"今天的状态"和"看到的云"
- 预测：明天可能会下雨

限制：你不知道明天实际发生了什么
```

**这就是Model**：
- 只能根据过去的状态和当前的信息（图像特征）来预测
- 符合真实的预测过程（你预测未来时，确实看不到未来）

---

### 方式2：Guide（变分后验）- "有标准答案的学习"

**特点**：训练时可以看到未来（因为有标准答案）

```
你现在知道：
- 今天天气：晴天
- 今天的温度：25度
- 你看到：天空有很多云

但是（训练时）你还有标准答案：
- 未来一周的天气记录：明天下雨，后天晴天，大后天阴天...

你可以结合"今天的状态"和"未来的标准答案"
来更准确地推断"今天应该是什么状态"

这样学习：如果今天状态是X，明天是Y，那我能学到什么规律？
```

**这就是Guide**：
- 训练时：可以看到未来的真实观测（因为有标注数据）
- 可以更准确地推断隐状态（因为知道"未来会发生什么"）
- **但推理时不能用**（因为没有未来的标准答案）

---

## 🔍 第二步：理解为什么需要两个？

### 核心问题：隐状态无法直接观测

**问题**：
- 我们只能观测到眼睛位置 `x_t`（可以测量）
- 但无法观测到隐状态 `z_t`（注意力状态，看不见）

**怎么办**？
- 需要从观测 `x_t` **推断**隐状态 `z_t`
- 但真实的后验分布 `p(z_t | x_t)` 很难计算
- **解决方案**：用 `q(z_t | x_t)`（Guide）来近似

---

### 变分推断的核心思想

```
真实情况（不知道）：
  p(z_t | x_t) = 真实的后验分布（很难计算）

训练时的近似（用Guide）：
  q(z_t | x_t) ≈ p(z_t | x_t)  （用神经网络近似）

目标：
  让q尽可能接近p（通过KL散度最小化）
```

**Guide的作用**：
- 训练时：近似真实的后验分布
- 帮助学习Model的参数
- **推理时不需要**（因为已经训练好了Model）

---

## 📊 第三步：对比Model和Guide

### 关键对比表

| 特性 | Model（生成模型） | Guide（变分后验） |
|------|-----------------|------------------|
| **数学表示** | `p(x_t \| z_t)`, `p(z_t \| z_{t-1})` | `q(z_t \| z_{t-1}, x_{t:T})` |
| **用途** | 生成新序列 | 训练时推断隐状态 |
| **转移组件** | GatedTransition | Combiner |
| **能看到未来** | ❌ 不能 | ✅ 可以（通过RNN） |
| **输入（预测z_t）** | `z_{t-1}` + 图像特征 | `z_{t-1}` + RNN编码的未来信息 |
| **训练时** | ✅ 使用 | ✅ 使用 |
| **推理时** | ✅ 使用 | ❌ 不使用 |

---

### 为什么Model只能用GatedTransition？

**原因**：Model模拟真实的生成过程

```
真实生成过程：
  时间步t时，你只能根据：
  - 过去的状态：z_1, z_2, ..., z_{t-1}
  - 图像信息：img_feature
  
  来预测：z_t

你不能"看到未来"，因为未来还没发生！
```

**所以Model**：
- 只能用GatedTransition（只看过去和图像）
- 符合真实世界的因果性

---

### 为什么Guide能用Combiner（看未来）？

**原因**：训练时有标准答案

```
训练时：
  我们有完整的序列：x_1, x_2, ..., x_T（标准答案）
  
  虽然未来"还没发生"，但我们有标注数据！
  可以用RNN处理反向序列，编码未来信息
```

**所以Guide**：
- 可以用Combiner（结合过去和未来信息）
- 训练时才能这样（因为知道标准答案）
- 推理时不能用（因为没有未来数据）

## 🔄 第四步：看代码理解实现

### Model的代码流程

```python
def model(self, scanpaths, ...):
    # 步骤1：提取图像特征
    img_features = self.cnn(images)  # (batch, 100)
    
    # 步骤2：初始化隐状态
    z_prev = 初始状态
    
    # 步骤3：对每个时间步循环
    for t in range(1, T_max + 1):
        # 3.1 用GatedTransition预测（只看过去和图像）
        z_mu, z_sigma = self.trans(z_prev, img_features)
        #                  ↑ GatedTransition（只看过去）
        
        # 3.2 采样隐状态
        z_t = pyro.sample("z_%d" % t, Normal(z_mu, z_sigma))
        
        # 3.3 用Emitter生成观测
        x_mu, x_sigma = self.emitter(z_t)
        
        # 3.4 观测（训练时用真实值，预测时采样）
        pyro.sample("obs_x_%d" % t, Normal(x_mu, x_sigma), 
                    obs=scanpaths[:, t-1, :])
        
        z_prev = z_t
```

**关键点**：
- ✅ 使用 `GatedTransition`：只看 `z_prev` 和 `img_features`
- ❌ 看不到未来的观测 `x_{t+1}, ..., x_T`
- ✅ 符合真实生成过程

---

### Guide的代码流程

```python
def guide(self, scanpaths, scanpaths_reversed, ...):
    # 步骤1：RNN处理反向序列（编码未来信息）
    rnn_output, _ = self.rnn(scanpaths_reversed)
    #                          ↑ 反向序列：从未来往过去看
    # rnn_output[:, t-1, :] 包含了x_t到x_T的信息
    
    # 步骤2：初始化隐状态
    z_prev = 初始状态
    
    # 步骤3：对每个时间步循环
    for t in range(1, T_max + 1):
        # 3.1 用Combiner推断（结合过去和未来）
        z_mu, z_sigma = self.combiner(z_prev, rnn_output[:, t-1, :])
        #                  ↑ Combiner（可以看到未来！）
        #                      z_prev: 过去
        #                      rnn_output: 未来信息
        
        # 3.2 采样隐状态（名称必须与model相同！）
        z_t = pyro.sample("z_%d" % t, Normal(z_mu, z_sigma))
        #                     ↑ 名称必须与model中的相同
        
        z_prev = z_t
```

**关键点**：
- ✅ 使用 `Combiner`：看 `z_prev` 和 `rnn_output`（未来信息）
- ✅ 可以看到未来的观测（通过RNN）
- ✅ **名称必须与model相同**（`"z_%d" % t`），这样Pyro才能匹配

## ❓ 第五步：回答关键问题

### 问题1：为什么Model只能看过去，Guide可以看未来？

**答案**：

**Model（生成模型）**：
- 模拟真实的生成过程
- 真实预测时，你只能看过去，不能看未来
- 所以Model必须遵循这个限制

**Guide（变分后验）**：
- 训练时有标准答案（完整的观测序列）
- 可以用RNN处理反向序列，编码未来信息
- 这样推断更准确，帮助学习更好的Model

**类比**：
- Model = 考试时答题（不能看答案）
- Guide = 复习时学习（可以看答案，学会规律）

---

### 问题2：为什么训练需要两个，推理只需要Model？

**训练时**：

```
1. Guide：从观测序列推断隐状态（有标准答案，可以用未来信息）
   → 得到：q(z_t | x_t)

2. Model：生成观测序列（只能看过去）
   → 得到：p(x_t | z_t)

3. 计算损失：
   - KL散度：KL(q || p) - 让Guide接近真实后验
   - 重构损失：预测和真实的差异
   
4. 更新参数（让两个网络都变好）
```

**为什么需要Guide？**
- Model的参数需要学习，但隐状态无法直接观测
- 需要用Guide来推断隐状态，然后计算损失
- Guide帮助Model学习（通过KL散度）

---

**推理时**：

```
1. 只需要Model：生成新序列
   - 输入：图像
   - 输出：预测的扫描路径
   
2. 不需要Guide：
   - Guide只在训练时用来推断隐状态
   - 推理时我们已经有了训练好的Model
   - 直接用Model生成即可
```

**类比**：
- 训练时：需要Guide（标准答案）来学习
- 推理时：只用Model（学到的知识）来预测

---

### 问题3：GatedTransition和Combiner的区别是什么？

| 特性 | GatedTransition | Combiner |
|------|----------------|----------|
| **使用场景** | Model（生成模型） | Guide（变分后验） |
| **输入** | `z_{t-1}` + `img_feature` | `z_{t-1}` + `h_rnn`（未来信息） |
| **能看到未来** | ❌ 不能 | ✅ 可以 |
| **功能** | 预测下一个状态（生成） | 推断当前状态（推断） |

**关键区别**：

1. **输入不同**：
   ```python
   # GatedTransition
   z_mu, z_sigma = trans(z_prev, img_feature)
   #                       ↑ 图像特征（静态）
   
   # Combiner
   z_mu, z_sigma = combiner(z_prev, h_rnn)
   #                       ↑ RNN编码的未来信息（动态）
   ```

2. **信息来源不同**：
   - GatedTransition：只看过去状态和图像（符合真实生成过程）
   - Combiner：看过去状态和未来观测（训练时的优势）

3. **目的不同**：
   - GatedTransition：生成下一个状态
   - Combiner：推断当前状态（利用未来信息更准确）

---

### 问题4：什么是变分推断？为什么需要Guide？

**变分推断**：

**问题**：
- 真实后验分布 `p(z_t | x_t)` 很难直接计算（需要复杂的积分）

**解决方案**：
- 用一个简单的分布 `q(z_t | x_t)` 来近似
- 让 `q` 尽可能接近 `p`（通过KL散度最小化）

**数学表述**：
```
目标：找到q，使得 KL(q(z|x) || p(z|x)) 最小

这等价于最大化ELBO：
  ELBO = E_q[log p(x|z)] - KL(q(z|x) || p(z))

其中：
  - E_q[log p(x|z)]：重构损失（让预测接近真实）
  - KL(q || p)：正则化项（让q接近p）
```

**为什么需要Guide？**
- Guide定义了 `q(z_t | x_t)`（变分后验）
- 用神经网络（Combiner + RNN）来学习这个分布
- 通过训练，让Guide尽可能接近真实后验
- 这样就能用Guide来推断隐状态，帮助训练Model

## 🎬 第六步：完整流程示例

### 训练时（同时使用Model和Guide）

```
输入：
  - 图像：360度图片
  - 观测序列：真实的扫描路径 x_1, x_2, ..., x_T

步骤1：Guide推断隐状态
  - RNN处理反向序列 → rnn_output（编码未来信息）
  - 用Combiner推断每个时间步的隐状态
  - 得到：q(z_1), q(z_2), ..., q(z_T)

步骤2：Model生成预测
  - CNN提取图像特征
  - 用GatedTransition + Emitter生成预测序列
  - 得到：p(x_1|z_1), p(x_2|z_2), ..., p(x_T|z_T)

步骤3：计算损失
  - KL散度：KL(q(z_t) || p(z_t))  （对每个时间步）
  - 重构损失：预测和真实的差异
  - 总损失 = KL散度 + 重构损失

步骤4：反向传播
  - 更新Model和Guide的参数
```

**关键点**：
- Guide推断隐状态（用未来信息，更准确）
- Model生成预测（只看过去，符合真实过程）
- 两者配合，让模型学习

---

### 推理时（只用Model）

```
输入：
  - 图像：360度图片
  - 没有观测序列（要生成）

步骤1：Model生成
  - CNN提取图像特征
  - 从初始状态开始
  - 每一步：
    * GatedTransition预测隐状态（只看过去和图像）
    * Emitter生成眼睛位置
  - 输出：完整的扫描路径

不需要Guide：
  - Guide只在训练时用
  - 推理时用训练好的Model即可
```

## 🧪 第七步：为什么Guide能看到未来？

### RNN处理反向序列的机制

```python
# 步骤1：反向序列
scanpaths_reversed = [x_T, x_{T-1}, ..., x_2, x_1]
#                      ↑ 从未来往过去看

# 步骤2：RNN处理
rnn_output, _ = rnn(scanpaths_reversed)
# rnn_output[:, i, :] 包含了从x_{i+1}到x_T的信息

# 步骤3：反转回正向顺序
rnn_output = reverse(rnn_output)
# 现在：rnn_output[:, t-1, :] 包含了x_t到x_T的信息
```

**例子**（假设T=5）：

| 时间步t | RNN处理的输入（反向） | RNN输出包含的信息 | 推断z_t时使用 |
|--------|---------------------|-----------------|-------------|
| 1 | x_5, x_4, x_3, x_2, x_1 | x_2到x_5的信息 | rnn_output[0] |
| 2 | x_5, x_4, x_3, x_2, x_1 | x_3到x_5的信息 | rnn_output[1] |
| 3 | x_5, x_4, x_3, x_2, x_1 | x_4到x_5的信息 | rnn_output[2] |

**关键理解**：
- RNN先处理完整序列（从未来往过去）
- 在推断时间步t的隐状态时，RNN已经"看到"了x_{t+1}到x_T
- 这就是"可以看到未来"的原因
