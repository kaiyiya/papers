# DMM (深度马尔可夫模型) 完整解释

## 🎯 整体目标

这个模型要做的事情：**根据360度图像，预测人们的视线扫描路径**

**比喻**：就像预测"一个人看这张图片时，眼睛会按照什么顺序看哪些地方"

---

## 🏗️ 模型架构概览

### 核心思想：隐状态模型

模型认为人的视线路径由**隐状态（隐藏的心理状态）**驱动：
- **隐状态 z_t**：你看图片时的"注意力状态"（100维向量，我们看不到）
- **观测 x_t**：实际的眼睛位置（3D坐标：x, y, z，我们可以观测到）

**关系**：
```
隐状态 z_t → 发射器 → 观测 x_t（眼睛位置）
     ↑              ↓
     └───转移───────┘
```

---

## 🧩 模型的主要组件

### 1️⃣ Sphere_CNN（球面CNN）
**作用**：从360度图像中提取特征
- 输入：360度图像
- 输出：图像特征（100维）
- **比喻**：从图片中提取"吸引人注意的地方"的信息

### 2️⃣ GatedTransition（门控转移）
**作用**：建模隐状态之间的转移 `p(z_t | z_{t-1})`
- 输入：前一个隐状态 `z_{t-1}` + 图像特征
- 输出：下一个隐状态的分布参数（mu, sigma）
- **比喻**：根据"刚才的注意力状态"和"图片内容"，预测"下一步注意力会变到哪里"

### 3️⃣ Emitter（发射器）
**作用**：从隐状态生成观测 `p(x_t | z_t)`
- 输入：隐状态 `z_t`
- 输出：眼睛位置的分布参数（mu, sigma）
- **比喻**：根据"当前的注意力状态"，预测"眼睛应该看哪里"

### 4️⃣ Combiner（组合器）
**作用**：变分后验 `q(z_t | z_{t-1}, x_{t:T})`
- 输入：前一个隐状态 `z_{t-1}` + RNN编码的未来信息
- 输出：隐状态的分布参数（mu, sigma）
- **比喻**：结合"过去的状态"和"未来的观测"，估计"当前应该是什么状态"

### 5️⃣ RNN
**作用**：处理反向序列，编码未来观测信息
- 输入：反向的观测序列 `x_T, x_{T-1}, ..., x_1`
- 输出：每个时间步的隐藏状态（编码了未来信息）
- **比喻**：像一个"时间倒流的观察者"，从未来往回看，总结未来会发生什么

---

## 🔄 两种模式：model 和 guide

### 模式1：model（生成模型）- "正向生成"

**目标**：模拟如何生成一条扫描路径

**流程**：
```
图像 → CNN → 图像特征
                ↓
初始状态 z_0 ───→ [时间步1] → z_1 → Emitter → x_1（预测的眼睛位置）
                ↓ GatedTransition
                [时间步2] → z_2 → Emitter → x_2
                ↓
                [时间步3] → z_3 → Emitter → x_3
                ...
```

**通俗理解**：
1. 看图像，提取特征
2. 从初始状态开始
3. 每一步：
   - 根据"上一步的状态"和"图像特征"，预测"下一步的状态"（GatedTransition）
   - 根据"当前状态"，预测"眼睛应该看哪里"（Emitter）
4. 生成完整的扫描路径

**用途**：生成新的扫描路径（预测模式）

---

### 模式2：guide（变分后验）- "反向推断"

**目标**：从观测到的扫描路径，推断隐状态

**流程**：
```
观测序列 x_1, x_2, ..., x_T
    ↓ 反向处理
RNN ← [x_T, x_{T-1}, ..., x_1]
    ↓
rnn_output（每个时间步编码了未来信息）
    ↓
初始状态 z_0 ───→ [时间步1] → z_1（结合z_0和RNN输出）
                ↓ Combiner
                [时间步2] → z_2（结合z_1和RNN输出）
                ↓
                [时间步3] → z_3（结合z_2和RNN输出）
                ...
```

**通俗理解**：
1. 先把观测序列反向输入RNN（从未来往过去看）
2. RNN编码每个时间步的"未来信息"
3. 从初始状态开始
4. 每一步：
   - 结合"上一步的状态"和"RNN编码的未来信息"，推断"当前的状态"（Combiner）
5. 得到所有时间步的隐状态

**用途**：训练时近似真实后验分布

### model方法（生成模型）

```python
def model(self, scanpaths, ...):
    # 步骤1：初始化隐状态
    z_prev = z_0  # 从初始状态开始
    z_prev = 合并(z_prev, 第一个观测)  # 结合初始状态和第一个观测
    
    # 步骤2：提取图像特征
    img_features = CNN(images)  # 图像 → 100维特征
    
    # 步骤3：对每个时间步循环
    for t in range(1, T_max + 1):
        # 3.1 计算转移分布（根据上一步状态和图像特征）
        z_mu, z_sigma = GatedTransition(z_prev, img_features)
        
        # 3.2 从转移分布采样隐状态 z_t
        z_t = 采样(Normal(z_mu, z_sigma))
        
        # 3.3 计算发射分布（根据当前隐状态）
        x_mu, x_sigma = Emitter(z_t)
        
        # 3.4 训练模式：提供真实观测（计算损失）
        #     预测模式：不提供观测（用于生成）
        if not predict:
            pyro.sample("obs_x_%d" % t, ..., obs=scanpaths[:, t-1, :])
        else:
            pyro.sample("obs_x_%d" % t, ...)  # 生成新样本
        
        z_prev = z_t  # 更新为当前状态，用于下一步
```

**时间步示例**（假设T=5）：

| 时间步 | 隐状态 | 转移过程 | 发射过程 | 观测 |
|--------|--------|---------|---------|------|
| 0 | z_0 | - | - | x_0（初始） |
| 1 | z_1 | GatedTransition(z_0, img) | Emitter(z_1) | x_1 |
| 2 | z_2 | GatedTransition(z_1, img) | Emitter(z_2) | x_2 |
| 3 | z_3 | GatedTransition(z_2, img) | Emitter(z_3) | x_3 |
| 4 | z_4 | GatedTransition(z_3, img) | Emitter(z_4) | x_4 |
| 5 | z_5 | GatedTransition(z_4, img) | Emitter(z_5) | x_5 |

### guide方法（变分后验）

```python
def guide(self, scanpaths, scanpaths_reversed, ...):
    # 步骤1：RNN处理反向序列
    rnn_output, _ = RNN(scanpaths_reversed)  # 从未来往过去看
    rnn_output = 反转(rnn_output)  # 转回正向顺序
    
    # 步骤2：初始化隐状态
    z_prev = z_0  # 从初始状态开始
    z_prev = 合并(z_prev, 第一个观测)
    
    # 步骤3：对每个时间步循环
    for t in range(1, T_max + 1):
        # 3.1 组合上一步状态和RNN输出（编码了未来信息）
        z_mu, z_sigma = Combiner(z_prev, rnn_output[:, t-1, :])
        
        # 3.2 从变分后验采样隐状态 z_t
        z_t = 采样(Normal(z_mu, z_sigma))
        
        z_prev = z_t  # 更新为当前状态
```

**关键区别**：
- **model**：使用 `GatedTransition`，只看过去和图像
- **guide**：使用 `Combiner`，可以看到未来信息（通过RNN）

**时间步示例**（假设T=5，RNN已经处理完整个序列）：

| 时间步 | 隐状态 | 组合过程 | RNN信息 |
|--------|--------|---------|---------|
| 0 | z_0 | - | - |
| 1 | z_1 | Combiner(z_0, rnn[0]) | rnn[0]编码了x_2到x_5的信息 |
| 2 | z_2 | Combiner(z_1, rnn[1]) | rnn[1]编码了x_3到x_5的信息 |
| 3 | z_3 | Combiner(z_2, rnn[2]) | rnn[2]编码了x_4到x_5的信息 |
| 4 | z_4 | Combiner(z_3, rnn[3]) | rnn[3]编码了x_5的信息 |
| 5 | z_5 | Combiner(z_4, rnn[4]) | rnn[4]只有当前信息 |

## 🔀 model vs guide 对比

| 特性 | model（生成模型） | guide（变分后验） |
|------|------------------|-------------------|
| **用途** | 生成新序列 | 训练时推断隐状态 |
| **转移组件** | GatedTransition | Combiner |
| **能看到未来** | ❌ 不能 | ✅ 可以（通过RNN） |
| **输入** | z_{t-1} + 图像特征 | z_{t-1} + RNN编码的未来信息 |
| **方向** | 正向生成 | 反向处理 + 正向推断 |

## 🎬 完整流程示例

### 训练时（同时使用model和guide）

```
1. 输入：
   - 图像：360度图片
   - 观测序列：真实的扫描路径 x_1, x_2, ..., x_T

2. guide（变分后验）：
   - RNN处理反向序列 → rnn_output
   - 用Combiner推断每个时间步的隐状态 z_1, z_2, ..., z_T

3. model（生成模型）：
   - CNN提取图像特征
   - 用GatedTransition + Emitter生成预测序列 x'_1, x'_2, ..., x'_T

4. 计算损失：
   - KL散度：guide和model的隐状态分布差异
   - 重构损失：预测序列和真实序列的差异
   - 总损失 = KL散度 + 重构损失

5. 反向传播更新参数
```

### 预测时（只用model）

```
1. 输入：
   - 图像：360度图片
   - 没有观测序列（要生成）

2. model（生成模型）：
   - CNN提取图像特征
   - 从初始状态开始
   - 每一步：
     * GatedTransition预测隐状态
     * Emitter生成眼睛位置
   - 生成完整的扫描路径
```

## 💡 关键设计理念

### 1. 为什么需要两个转移组件？

- **GatedTransition（model）**：模拟真实生成过程（只能看过去）
- **Combiner（guide）**：训练时的近似推断（可以看到未来）

**比喻**：
- GatedTransition = 真实的预测（你预测明天，只能根据今天）
- Combiner = 训练时的"作弊"（你有标准答案，可以看未来来学习）

### 2. 为什么RNN要处理反向序列？

因为要编码"未来信息"。如果正向处理，在时间步t时，RNN只看到x_1到x_t，看不到x_{t+1}到x_T。

**反向处理的优势**：
- 时间步t的RNN输出已经"看到"了x_{t+1}到x_T
- 这样Combiner在推断z_t时，可以利用未来信息

### 3. 变分推断的作用

真实的隐状态后验 `p(z_{1:T} | x_{1:T})` 很难直接计算，所以用 `q(z_{1:T} | x_{1:T})`（guide）来近似。

**训练目标**：让guide尽可能接近真实后验，同时让model能生成好的观测。

## 🎯 总结

**DMM模型的核心**：
1. **隐状态模型**：观测由隐状态驱动
2. **两种模式**：
   - model：正向生成（只能看过去）
   - guide：反向推断（训练时看未来）
3. **四个关键组件**：
   - CNN：提取图像特征
   - GatedTransition：状态转移（生成）
   - Combiner：变分后验（推断）
   - Emitter：从状态生成观测
4. **训练目标**：学会从图像预测扫描路径

**一句话理解**：模型学习"看到图片时会怎么移动视线"，通过隐状态来建模这个过程。
