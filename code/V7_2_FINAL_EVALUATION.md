# V7.2模型最终评估报告（修正序列长度）

**生成时间**: 2026-02-05
**模型**: V7.2 Best Model (Epoch 49)
**修正内容**:
1. 使用正确的12×8网格离散化（LEV）
2. 使用预测与GT的交叉递归（REC）
3. **根据GT原始长度截断预测序列进行评估**

---

## 🔧 关键修正

### 问题发现
不同数据集的scanpath长度不同：
- **AOI**: 9个点
- **JUFE**: 15个点
- **Salient360**: 25个点
- **Sitzmann**: 30个点

之前的评估将所有序列padding到25个点，导致：
- AOI用25个点的预测对比9个点的GT → LEV异常高（36.45）
- JUFE用25个点的预测对比15个点的GT → LEV异常高（35.92）

### 修正方法
在评估时，只使用GT的原始长度进行对比：
- AOI: 只比较前9个点
- JUFE: 只比较前15个点
- Salient360: 比较前25个点
- Sitzmann: 比较前30个点（但模型只生成25个，所以实际比较25个）

---
